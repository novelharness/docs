# 模块边界与角色职责矩阵

## 1. 角色层（谁负责什么）

| 角色 | 核心职责 | 产出物 | 不负责 |
|---|---|---|---|
| 产品负责人（你） | 定义题材、目标读者、商业节奏 | 创作方向、优先级 | 技术实现细节 |
| 系统架构师 | 定义运行时与模块边界 | 架构文档、接口契约 | 逐章内容质量 |
| 世界观策划 Agent | 维护静态设定 | 世界观条目、规则集 | 临场文风润色 |
| 大纲规划 Agent | 卷纲/细纲拆解 | 章节任务清单 | 最终行文品质 |
| 正文生成 Agent | 章节草稿生成 | 草稿正文 | 设定裁决 |
| 质量守门 Agent | 质检与修改建议 | 质检报告、修订点 | 业务排期 |
| 记忆管理员 Agent | 摘要压缩与索引维护 | 记忆快照、检索键 | 主线创意决策 |

## 2. 模块边界（系统层）

## A. Novel Domain Layer（小说领域层）

- 定义：剧情规则、人设规则、节奏规则。
- 只输出规则与策略，不直接调用底层模型。

## B. Runtime Orchestration Layer（运行时编排层）

- 定义：任务状态机、Query Loop、Hook 流程。
- 负责“怎么跑”，不负责“写什么”。

## C. Infrastructure Layer（基础设施层）

- 定义：存储、检索、日志、权限。
- 提供稳定能力，不承载业务判断。

## 3. 章节任务状态机（建议）

`pending -> context_loaded -> drafted -> reviewed -> revised(optional) -> approved -> persisted -> compacted`

异常支线：
- `review_failed -> revised -> reviewed`
- `persist_failed -> retry_or_resume`

## 4. 关键接口契约（文档级）

### 4.1 `GenerateChapter(input)`

- 输入：
  - `chapter_goal`
  - `required_events`
  - `context_bundle`
- 输出：
  - `chapter_draft`
  - `state_delta_candidate`
  - `foreshadow_changes`

### 4.2 `ReviewChapter(input)`

- 输入：`chapter_draft`, `ruleset`, `previous_state`
- 输出：
  - `pass:boolean`
  - `issues[]`
  - `rewrite_suggestions[]`

### 4.3 `CommitChapter(input)`

- 输入：`final_text`, `final_state_delta`, `metadata`
- 输出：`commit_id`, `snapshot_id`

## 5. 权限分层（借鉴 runtime deny-first）

- **L0 只读**：设定库、已发布章节。
- **L1 受控写**：草稿区、分析报告区。
- **L2 审批写**：正式章节库、状态主表（需质量门禁通过）。

## 6. 冲突处理机制

- 冲突类型：
  1. 人设冲突
  2. 战力冲突
  3. 时间线冲突
  4. 伏笔冲突
- 冲突处理顺序：
  - 自动修复建议 → 人工确认 → 回滚最近提交并重跑。

## 7. 角色与模块协作原则

1. 角色说“业务语言”，模块说“系统语言”。
2. 所有跨模块通信都结构化（禁止口语化隐式协议）。
3. 所有可写操作都可回滚。
