# 实施计划（按服务拆解）

## 1. 迭代 0：文档冻结（1 周）

- 冻结 API 契约（Go）
- 冻结配置模型（Control Plane）
- 冻结 Runtime 输入输出协议
- 冻结 n8n 工作流模板（WF-01~WF-06）

产出：ADR + OpenAPI + ERD + Workflow Spec。

## 2. 迭代 1：Go 主服务 MVP（2~3 周）

- 小说/章节/世界观/伏笔 CRUD
- Runtime 提交接口（chapter-commit）
- 审计与版本回滚

验收：可手工跑通一章提交闭环。

## 3. 迭代 2：Runtime MVP（2 周）

- 接入 Query Loop
- 接入 context bundle
- 产出草稿 + 质检报告 + state_delta

验收：连续 5 章生成，状态一致。

## 4. 迭代 3：n8n 主流程上线（1~2 周）

- 上线 WF-02（章节生产主流程）
- 加人工审核分支
- 加失败重试与告警

验收：流程自动化率 > 70%。

## 5. 迭代 4：配置中心上线（2 周）

- Agent/Skill/MCP/Model 管理
- 配置发布与版本追踪
- Runtime 拉取配置快照

验收：配置切换可灰度、可回滚。

## 6. 迭代 5：前端工作台（2 周）

- 小说项目管理
- 章节工作台
- 工作流与任务运行看板
- 平台配置页（模型/Agent/MCP）

验收：编辑与运营可在 web 完成主要操作。
