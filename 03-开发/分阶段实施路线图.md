# 分阶段实施路线图（先文档、后实现）

## 0. 实施总原则

- 每阶段只解决一类核心风险。
- 每阶段都有“输入、动作、输出、验收”。
- 未达标不进下一阶段。

## Phase 1：MVP 骨架（可稳定产出 1 章）

### 目标

实现最小闭环：`上下文加载 -> 草稿生成 -> 质检 -> 提交 -> 记忆压缩`。

### 范围

- 单主角、单主线。
- 单 Agent 生成 + 单 Agent 审核。
- 手工触发流程（先不全自动）。

### 输出

- 章节任务模板
- 章节状态模板
- 质检模板
- 章节提交模板

## Phase 2：可持续生产（可连续产出 10~30 章）

### 目标

把“能跑”升级为“持续稳定跑”。

### 增量能力

- Session 恢复
- 自动重试与失败回滚
- 摘要检索与上下文重建
- 节奏监控看板（基础）

### 输出

- 恢复策略文档
- 失败注入清单
- 节奏偏差报告模板

## Phase 3：多 Agent 协作（质量提升）

### 目标

引入多角色分工，降低单模型输出抖动。

### 增量能力

- 大纲规划、正文生成、节奏审查并行候选
- 主控合并策略（投票/打分/规则优先）
- 角色一致性自动巡检

### 输出

- 多 Agent 协作规约
- 候选合并规则
- 冲突优先级策略

## Phase 4：运营驱动优化（商业化能力）

### 目标

把读者反馈数据纳入下一章生成策略。

### 增量能力

- 反馈信号接入（追读、评论、停留）
- 自动调参（节奏、冲突密度、角色出场占比）
- 版本实验（A/B 章节策略）

### 输出

- 反馈特征字典
- 调参与回归评估流程
- 运营周报模板

## 附：阶段切换门槛

- P1→P2：连续 5 章无严重设定冲突。
- P2→P3：恢复成功率 >= 95%，平均返工轮次 <= 1.5。
- P3→P4：多 Agent 策略相较单 Agent 质量评分提升 >= 15%。
