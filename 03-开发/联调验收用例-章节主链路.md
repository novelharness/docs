# 联调验收用例：章节主链路（强一致）

## Case 1：标准通过链路

- 前置：章节 N-1 已提交。
- 步骤：触发 workflow -> 自动门禁通过 -> 自动审批通过 -> commit。
- 预期：
  - 章节发布成功
  - world/character/foreshadow 同步更新
  - task_run = DONE

## Case 2：并发冲突

- 步骤：同时触发同一 `novel_id + chapter_no` 两次。
- 预期：
  - 仅一个实例拿到 lock
  - 另一个实例返回 `LOCK_CONFLICT`

## Case 3：自动门禁失败

- 前置：构造 critical conflict。
- 预期：
  - 返回 `QUALITY_GATE_FAILED`
  - 不发生 commit
  - 可进入修订分支

## Case 4：人工审批拒绝

- 前置：强制人工审批。
- 预期：
  - 状态进入 REJECTED
  - 回到 Editor 回修
  - 未写入正式章节

## Case 5：提交事务失败

- 前置：模拟 DB 异常。
- 预期：
  - 返回 `COMMIT_TX_FAILED`
  - 全量回滚
  - 锁释放

## Case 6：后处理失败

- 前置：模拟索引服务不可用。
- 预期：
  - 章节已提交（COMMITTED）
  - 主流程状态 `DONE_WITH_WARNING`
  - 生成补偿任务

## Case 7：流程中断恢复

- 前置：运行到审校后停止。
- 步骤：resume。
- 预期：
  - 从中断节点恢复
  - 最终成功提交
