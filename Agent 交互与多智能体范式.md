# Agent 交互与多智能体范式

## 背景

在小说工程化创作中，多个 Agent 必须协同工作才能完成从设定到输出的复杂流程。正确的交互模式可以减少上下文冲突、提高效率并提升最终输出质量。本文基于 Claude Code 工作流模式和多智能体设计经验，介绍适合小说创作的交互形式。

## Claude Code 的多种工作流模式

Claude Code 文档将 Agent 工作流划分为五种模式：Sequential Flow、Operator (Orchestrator-Subagent)、Split-and-Merge、Agent Teams 和 Headless。每种模式适用于不同场景。

- **Sequential Flow（顺序流水线）**：任务按固定顺序执行，每一步的输出作为下一步的输入。适用于结构明确、线性依赖的流程。
- **Operator (Orchestrator-Subagent)**：一个主控 Agent 负责规划和调度，其它子 Agent 执行具体任务，主控合并结果。适用于需协调多工具的复杂任务。
- **Split-and-Merge（拆分与合并）**：将大量独立任务分散至多个子进程并行执行，最后由合并逻辑汇总。适用于批量处理或需要投票/共识的场景。
- **Agent Teams（团队协作）**：多个具有独立思考能力的 Agent 协同完成共同目标，可通过管道、同行评审或专家分工模式实现。适用于需要多个专长并需要质量审核的任务。
- **Headless（全自动）**：无人工干预，预设触发后自动运行整个流程。适用于完全成熟、风险可控的流程。

## 适用于小说创作的交互模式

结合小说创作特点，推荐以下模式或组合：

1. **管道型团队（Pipeline Team）**
   - **世界设定 Agent → 大纲规划 Agent → 剧情生成 Agent → 节奏分析 Agent → 质量守门员 Agent**
   - 每个 Agent 专注自身子任务，完成后将结果传递给下一个。
   - 利用 **Sequential Flow** 保证线性依赖；在节奏分析和质量控制处如有问题可反馈回前一步重新生成，形成小范围的循环。
2. **专家团队（Specialist Team）**
   - 在剧情生成阶段可并行运行多个不同风格的生成 Agent（如冒险风格、悬疑风格），然后由评审 Agent 选择或合并最佳版本。
   - 结合 **Split-and-Merge** 模式：多个 Agent 同时创作不同版本的支线或冲突场景。
3. **同行评审（Peer Review Team）**
   - 故事生成后，由另一名审稿 Agent 按设定与节奏标准评审，并提出修改意见。指出，通过多 Agent 交叉评审可提升质量。
   - 这种模式适用于关键剧情节点或篇章结尾，防止逻辑漏洞。
4. **角色模拟互动**
   - 为故事中的主要角色设计独立的“角色 Agent”，它们在同一个世界观设定下拥有自己的性格、目标和记忆。
   - 这些角色 Agent 在虚拟环境中对事件作出反应，推动情节发展。导演 Agent 充当 DM（Dungeon Master），负责提供场景和冲突，并引导角色朝着预定的主线或支线收敛。
   - 每个角色 Agent 具备自己的上下文窗口，存储个人经历和关系；在做出决策时只引用相关记忆，避免跨角色污染。
   - 角色交互可以通过事件传递机制实现：导演 Agent 发布事件，角色 Agent 返回行为与对话，导演再整合这些结果形成剧情。
5. **Headless 周期**
   - 系统运行一段时间后（例如夜间批量创作），可以进入 headless 模式，自动生成若干章节并提交节奏分析与质量守门员。只在出现错误或异常时通知人工审核。

## Agent 间通信形式

- **消息队列**：通过 RabbitMQ/Redis Stream 传递消息。每个 Agent 订阅自己的任务队列并发布结果到下游队列。
- **共享数据库**：将结果写入数据库表，下游 Agent 通过轮询或触发器读取新记录。
- **HTTP/Webhook**：使用 REST 或 Webhook 调用其他 Agent 服务，在微服务架构中常见。
- **文件系统**：对于初级实现，可通过写入/读取文件交换数据，但不适合并发场景。

采用消息队列加数据库的混合模式可以兼顾解耦和持久化：任务和事件通过队列流动，持久化数据存储在数据库中，便于恢复和分析。

## 角色活化与元宇宙体验

为了让角色看起来“鲜活”，不仅需要合理的情节，还要赋予角色 Agent 内在动力和个性：

- **性格模型**：为每个角色定义性格标签（如忠厚、腹黑、冲动），在角色 Agent 的系统提示中体现。决策时结合性格倾向。
- **动机与目标**：设置角色的长期目标（如复仇、追求力量）与短期目标（如完成试炼）。导演 Agent 应在设计剧情时为角色提供“钩子”刺激其行动。
- **信息局限**：每个角色 Agent 只能访问其视角内的信息（例如不知道其他角色的秘密），这样他们的行为更真实。
- **反馈机制**：导演 Agent 根据角色行为修改世界状态（成功/失败/受伤），角色 Agent 根据结果调整策略。

通过这种元宇宙式的 Agent 交互，故事可以既遵循总体框架，又保留角色的自主性和偶然性，增加故事张力。

## 总结

多智能体系统需要选择合适的工作流模式和通信机制来匹配任务需求。小说创作适合使用管道型和专家团队模式，并辅以拆分与合并来探索多种剧情可能性。通过消息队列和数据库提供可靠的交互通道，并为角色 Agent 设定性格、目标和信息局限，使其在真实世界观中自主行动，最终在导演 Agent 的调度下收敛到预定框架。这种系统既确保工程化的可控性，又保留文学创作的灵魂。
