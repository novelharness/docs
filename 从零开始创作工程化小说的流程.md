# 从零开始创作工程化小说的流程

## 引言

工程化创作小说并非一蹴而就，它是一系列小工程的组合，需要规划、执行、监控和迭代。之前的文档分别介绍了存储方案、智能体设计、持久化运行、记忆管理、交互范式、任务拆解、动态世界管理、章节工程化和对话设计。为便于使用，这里将各部分汇总为从概念到完成小说的完整流程，帮助创作者一步一步搭建起 AI 协作创作系统。

## 阶段总览

1. **立项与核心设定**：确定故事的核心卖点、金手指和世界观基础。
2. **静态设定建模**：将世界观、人物档案和主线任务编制成结构化设定库，存储于数据库或本地文件系统。
3. **系统搭建与 Agent 定义**：配置调度中心（如 n8n）、部署智能体 Hub 和常驻智能体运行时，并根据《agent_design.md》定义各类智能体。
4. **动态世界状态设计**：设计数据结构跟踪世界和角色的动态变化，参考《dynamic_world.md》中的示例模型。
5. **大纲生成**：通过设定架构师和大纲规划师生成总纲、卷纲和细纲，形成可执行的任务拆解。
6. **章节创作流水线**：按照《chapter_engineering.md》设计每章，并调用 Story Generator 等智能体生成正文；根据《dialogue_design.md》设计对话细节。
7. **状态更新与循环**：章节生成后，自动解析正文，更新动态世界状态和记忆，并调整后续大纲或节奏。
8. **质量控制与迭代**：由节奏分析师和质量守门员进行评估和润色，必要时回滚修改大纲或提示词。

下面将各阶段拆解为具体步骤和执行建议。

## 1 立项与核心设定

### 1.1 提炼核心创意

确定一句话故事（电梯稿），明确目标受众和核心爽点。根据商业推导，评估主角金手指的功能、限制和成长路径（参考《storage.md》和《agent_design.md》）。

### 1.2 世界观与力量体系

建立世界的自然法则、社会结构和力量体系。确保内部一致性，避免矛盾；为主要人物设定动机和愿望，使他们的行为符合背景。

## 2 静态设定建模

将上述设定转换为结构化数据并存入设定库：

- **世界观表**：记录地理区域、宗门、等级、历史等。
- **金手指表**：描述功能、触发条件、消耗、升级路径和限制。
- **角色档案表**：记录姓名、身份、初始境界、性格标签、背景、长短期目标等。
- **主线任务表**：列出故事主线目标、关键节点和胜利条件。

可使用本地 JSON 文件、CSV 或数据库，如 PostgreSQL/MySQL（详见《storage.md》）。

## 3 系统搭建与 Agent 定义

### 3.1 部署调度中心

选择 n8n 作为工作流管理器，在其中编排各 Agent 的调用顺序、条件和错误处理。参考《agent_interaction.md》了解可用的多智能体范式（流水线模式、专家协同、同行评审、角色模拟等），根据项目规模选择合适架构。

### 3.2 配置智能体 Hub

搭建智能体仓库，用于存放智能体的静态定义、提示词、脚本和技能声明（见《agent_design.md》）。Hub 应支持版本管理，以便迭代优化。

### 3.3 启动常驻智能体

按照《persistent_agents.md》部署常驻智能体，如 Memory Manager（负责长期记忆和摘要）、World State Manager（维护动态世界数据）、Outline Planner、Story Generator 等。为每个智能体指定独立线程或容器，并制定健康监控机制。

## 4 动态世界状态设计

采用《dynamic_world.md》中的数据结构，每个章节前后更新角色状态、关系网、任务进度、环境状态、物品信息和信息差。确保系统能够区分“世界真相”和“角色认知”。

实现要点：

- 定义一个 `world_state.json` 或数据库表，按章节索引存储世界各元素的动态信息。
- 在创作流程中，章节生成后由解析模块自动更新状态，包括HP、境界、知识、关系等。
- 设计摘要压缩策略，将事件历史和伏笔信息提炼成精简文本供上下文调用，避免长篇记忆堵塞模型。

## 5 大纲生成

1. **读取设定库**：由设定架构师 Agent 加载静态设定，为大纲生成提供约束。
2. **生成总纲**：大纲规划师 Agent 基于起承转合模式编写全书蓝图，明确最终目标、金手指终极形态和最终反派。
3. **分卷拆解**：将总纲分解为多卷，每卷具有独立冲突、登场人物、战力变化和节奏节点，预设换图与高潮。
4. **生成细纲**：对每卷按照章节粒度制定工程化细纲，字段可参考《chapter_engineering.md》中的表格。确保涵盖目标、冲突、节奏划分、伏笔和钩子。
5. **审核和调整**：人为审核并修改大纲中的逻辑问题、节奏失衡或人物 OOC 风险，然后存入大纲库。

## 6 章节创作流水线

### 6.1 前置准备

- **加载上下文**：从动态世界状态和设定库取得最新信息，包括角色状态、环境、任务进度、未解决的伏笔。
- **确定章节目标**：依据细纲确定本章需完成的任务和核心事件。
- **规划节奏与爽点**：按照“三章一小爽、十章一大爽”的原则设计情绪曲线，并确定压抑与释放的比例。
- **设计伏笔与悬念**：参考伏笔表埋设新的伏笔，记录预期回收章节；在章末制造悬念，吸引读者阅读下一章。
- **设计对话**：使用《dialogue_design.md》中的模板，为关键对话制定目的、参与者和潜台词，并加入动作和环境描述。

### 6.2 生成与评估

1. **剧情生成**：Story Generator Agent 根据细纲和上下文生成正文草稿，包括对话、叙述、战斗等。
2. **节奏分析**：节奏分析师 Agent 评估草稿的情绪曲线和爽点分布，提出修改建议如调整压抑长度或增加爽点。
3. **质量守门**：质量守门员 Agent 检查逻辑一致性、角色是否 OOC、语言是否符合要求，并进行适度润色。
4. **人工审核**（可选）：人类作者查验故事走向、对话自然度和情感达成度，做适当修改。
5. **发布并更新状态**：将最终稿发布到剧情库；同步更新动态世界状态和记忆系统，包括 HP、关系值、任务进度和伏笔回收情况。
6. **调整大纲**：根据节奏分析和读者反馈（如数据追读率）适当调整后续细纲或提示词。

## 7 状态更新与循环

章节完成后，使用自动解析工具提取以下信息更新世界状态：

- **角色层面**：境界提升、HP 变化、词条获得或消耗、情绪和动机变化。
- **关系层面**：增减亲密度或敌对值，记录结盟或背叛。
- **任务层面**：更新主线和支线任务进度、解锁新任务或完成旧任务。
- **物品层面**：法宝、丹药等物品增减，更新库存。
- **事件历史**：记录发生的冲突、发现的秘密或埋下的新伏笔。

同时生成摘要，以简洁的叙述（约 50–100 字）概述本章内容，并提取关键词，供后续章节调用和记忆剪枝。

## 8 质量控制与迭代

- **节奏监控**：持续监控阅读数据（如追读率、停留时间），结合节奏分析师反馈调整大纲和节奏。
- **毒点排查**：避免出现常见毒点，如主角降智、无意义的虐心、刻意水字数；保持角色行为与性格一致，遵守底线要求（见《chapter_engineering.md》）。
- **提示词迭代**：根据质量守门员输出和读者反馈，持续优化各智能体的提示词，使输出更加精确。
- **硬件与性能优化**：确保系统可伸缩，包括数据库读写效率、模型调用速度和内存管理。

## 结论

以上流程将所有文档中的概念和工具串联起来，形成一个闭环的工程化小说创作体系。从立项到章节生成再到状态更新和反馈迭代，每一步都有明确的目标和责任主体。通过这种分层、结构化、可追踪的流程，你可以让多智能体协同工作，把一个宏大的小说项目拆解为可管理的任务，同时保持创意的连贯性和世界的动态真实性。
