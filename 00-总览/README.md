# AI 小说工程文档总览（v2）

## 1. 总体目标

把你的平台设计收敛成可执行体系：

- **Go + MySQL** 作为业务真相层（小说域 CRUD 与状态管理）
- **Claude Code 能力双服务拆分**（配置中心 + Runtime）
- **n8n** 作为核心业务编排层
- **Next.js** 作为创作与平台管理前端

## 3. 当前文档分层

- **01-调研**：Claude Code runtime 能力复用与边界分析。
- **02-设计**：系统蓝图、服务边界、数据契约、workflow 方案。
- **03-开发**：按服务和阶段推进的实施计划与验收标准。

## 4. 设计原则（固定）

1. **业务真相只在 Go API + MySQL**。
2. **n8n 只编排，不承载业务主存储**。
3. **Runtime 负责智能执行，不直接改主库**。
4. **配置中心与运行时分离，配置必须版本化**。
5. **所有关键动作可追溯、可回滚、可审计**。
