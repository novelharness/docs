# 冲突处理说明（本次 PR）

如果你在合并 `work -> main` 时出现冲突，按以下步骤处理：

1. 确保拉取最新 main。
2. 在 feature 分支执行：
   - `git merge main`（或 `git rebase main`）
3. 若冲突集中在文档：
   - 先保留主分支结构
   - 再手工合并本分支新增段落：
     - Agent 资产落地章节
     - Phase-0 初始化前置章节
4. 运行检查：
   - `rg -n "<<<<<<<|=======|>>>>>>>" .`
5. 再提交冲突解决结果。

本仓库已新增 `.gitattributes`，对高频冲突文档启用 `merge=union`，降低后续冲突概率。
