# 开发防跑偏清单（DoR / DoD）

## 1. DoR（开始开发前必须具备）

## 1.1 需求层

- [ ] 章节主流程状态机已冻结。
- [ ] Agent 参与清单与职责已冻结。
- [ ] 自动门禁阈值已确定。

## 1.2 契约层

- [ ] OpenAPI 已冻结（至少 v1）。
- [ ] `章节工作流输入输出契约` 已评审通过。
- [ ] 错误码与幂等规则已统一。

## 1.3 数据层

- [ ] MySQL ERD 已冻结。
- [ ] 事务边界明确。
- [ ] 锁与并发策略明确。

## 1.4 运维层

- [ ] trace/log/metrics 字段定义完毕。
- [ ] 告警策略定义完毕。
- [ ] 环境变量与密钥管理方案确定。

## 2. DoD（功能完成必须满足）

## 2.1 功能正确性

- [ ] 同章节并发提交只有一个成功。
- [ ] 强制人工审批章节无法绕过审批。
- [ ] 自动门禁不通过时不会落库。

## 2.2 一致性

- [ ] `chapter-commit` 事务原子。
- [ ] 提交成功时状态联动更新完整。
- [ ] 提交失败时无脏数据残留。

## 2.3 可恢复性

- [ ] 节点失败支持重试。
- [ ] 流程中断支持 resume。
- [ ] 后处理失败会生成补偿任务。

## 2.4 可观测性

- [ ] 全链路 trace 可检索。
- [ ] 每次失败有 error_code + stage。
- [ ] 指标看板可见：成功率、时延、返工率。

## 2.5 安全性

- [ ] 敏感接口有 RBAC。
- [ ] 服务间调用签名校验通过。
- [ ] 所有关键写操作留审计日志。
