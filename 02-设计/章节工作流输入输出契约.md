# 章节工作流输入输出契约（可直接给开发对齐）

## 1. 统一对象定义

## 1.1 WorkflowInput

```json
{
  "trace_id": "string",
  "novel_id": "string",
  "chapter_no": 12,
  "writing_goal": "string",
  "force_human_review": false,
  "operator": "user_id_or_system"
}
```

## 1.2 ContextBundle

```json
{
  "novel_meta": {},
  "world_static": {},
  "world_dynamic": {"version": 27},
  "arc_outline": {},
  "chapter_outline": {},
  "open_foreshadows": [],
  "character_state": [],
  "constraints": {
    "must_events": [],
    "forbidden_events": [],
    "tone": "热血"
  }
}
```

## 1.3 RuntimeOutput

```json
{
  "draft_text": "string",
  "quality_report": {
    "consistency_score": 90,
    "critical_conflicts": 0,
    "foreshadow_violations": 1,
    "sensitive_hits": 0
  },
  "state_delta": {
    "character_delta": [],
    "world_delta": [],
    "foreshadow_delta": []
  },
  "needs_human_review": false
}
```

## 2. 节点输入输出契约

## Node A：Lock & Preconditions

- In: `WorkflowInput`
- Out:
  - 成功：`lock_token`, `precheck_report`
  - 失败：`error_code=PRECHECK_FAILED`

## Node B：Build Context

- In: `novel_id`, `chapter_no`, `trace_id`
- Out: `ContextBundle`

## Node C：Run Runtime

- In: `ContextBundle + writing_goal`
- Out: `RuntimeOutput`

## Node D：Auto Gate

- In: `RuntimeOutput.quality_report`
- Rule:
  - `consistency_score >= threshold`
  - `critical_conflicts == 0`
  - `sensitive_hits == 0`
- Out:
  - `PASS` -> Commit
  - `REVIEW_REQUIRED` -> Human Review
  - `REWRITE_REQUIRED` -> Editor 回修

## Node E：Human Review

- In: `draft_text + quality_report + precheck_report`
- Out: `APPROVE | REJECT`

## Node F：Commit

- In: `final_text + state_delta + trace_id + config_version + lock_token`
- Out: `commit_id`, `chapter_version`, `snapshot_version`

## Node G：Post Process

- In: `commit_id`
- Out: `summary_id`, `index_status`, `metrics_report_id`

## 3. 错误码规范（最小集）

- `PRECHECK_FAILED`
- `LOCK_CONFLICT`
- `RUNTIME_TIMEOUT`
- `QUALITY_GATE_FAILED`
- `HUMAN_REVIEW_REJECTED`
- `COMMIT_TX_FAILED`
- `POST_PROCESS_FAILED`

## 4. 幂等规范

- 幂等键：`trace_id + chapter_no + node_name`
- Commit 节点幂等：同 `trace_id` 重试返回同一 `commit_id`

## 5. 回滚规范

- Commit 前失败：释放锁，任务状态 `FAILED`
- Commit 失败：事务回滚，释放锁，记录 `COMMIT_TX_FAILED`
- Commit 后失败：主流程 `DONE_WITH_WARNING`，创建补偿任务
